Your Data Inventory

Data Owners taking ownership of data

Review

  • Data Owners
    • Define access rules about your data (e.g., who should be able to view what data that you own)
    • Approve business rules for your data
    • Approve change requests from others related to your data
    • Submit change requests from others related to your data
  • Data Stewards
    • Maintain subject matter expertise about your data elements
    • Monitor regulatory and legislative requirements related to the collection, storage, and measurement of your data
    • Serve as the authoritative source of truth when there are questions related to your data
    • Interact with end users about questions related to your data
  • Data Managers
    • Implements your business rules in the data systems
    • Implements approved change requests
    • Monitors the data systems to ensure access is provided to approved users only
    • Works with data stewards to reconcile issues related to data quality assurance and quality control (QA/QC)
  • Long term goal is to integrate ALL FCPS data in data warehouse
  • Provides a one-stop shop for everyone to access data needed for decision making and reporting
  • Integrates all of the systems of records to avoid confusion
  • The data warehouse becomes the data source used for future program evaluations, cost-benefit analyses, research, dashboards, etc...
  • This does not change any existing functionality with any of the systems
  • State reports will still continue to operate exactly as they do now
  • Data will still be input into the systems exactly the way they do now

The Data Inventory

  • The list on SharePoint only contains info from IC and is based on the data model
    • This means you won't see specific tests, but will see the structures used to store test data
    • There are other data that aren't captured here, but it provides the starting point to map data governance roles to the data in the data warehouse
  • The file on SharePoint lists 1,585 different data elements from IC
  • There are 5,551 data elements in the data warehouse
  • As we integrate other data into the data model the 5,551 will quickly grow/expand
  • Being able to consistently and correctly identify data owners and stewards helps these processes work more efficiently
  • The data inventory makes it easier for us to communicate the data available for requests to end users
  • It also provides us with a way to think about routing questions based on the topics related to the data
  • It makes controlling the flow of change requests easier and more consistent to manage
  • For example, send questions about course names to the courses data steward and questions about assessment to Peggy
  • For example, if only data owners can submit change requests data managers will know whether or not the request has already been approved by the data owner
  • There are still data elements from the file on SharePoint that do not include the necessary owners/steward information
  • If we can identify data owners we can follow up with them individually regarding who would steward their data
  • This information will get mapped to the data warehouse and we will be able to build out a report to show who the data owners, stewards, and managers are for all the data elements in the data warehouse are in the near future
  • This is especially critical as we start the quality assurance/quality control work with the data warehouse

The Point

  • We need strong data governance because a single seemingly minute issue can have fairly significant ramifications in systems this complex
  • We need everyone to monitor data quality issues
  • We need data stewards to communicate the rules to people performing data entry
  • We need data owners to help motivate others to correct data quality issues when they come up
  • We need data managers to enforce the approved business rules
